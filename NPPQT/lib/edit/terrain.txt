# File: terrain.txt



# This file is used to initialize the 
# 'terrain feature' information for the NPPAngband and NPPMoria games.

# Do not modify this file unless you know exactly what you are doing,
# unless you wish to risk possible system crashes and broken savefiles.

# Note that terrain feature zero contains the 'DARKNESS' picture.

# === Understanding terrain.txt ===

# G: symbol : color {alternatively} G: symbol : red : green : blue
# M: feature to mimic
# E: edge with feature
# W: depth : rarity : priority : power : timeout
# C: damage to non-native: native energy move : non-native energy move :
#          native to hit adj : non-native to hit adj : stealth_adjust
# K: action name : change with feature : change chance
# F: flag | flag | etc
# D: Description
# 'G' is for graphics - symbol and color. Colors can be entered in
# red-green-blue format (values from 0-255, or any of the preset
# colors below may be used.  

#TERM_DARK is defined as 0-0-0 and nothing will appear onscreen

# TERM_DARK - {see note above - should not be used
# TERM_L_DARK
# TERM_SLATE
# TERM_SLATE_GRAY
# TERM_TAUPE
# TERM_WHITE
# TERM_LIGHT_GRAY
# TERM_SNOW_WHITE
# TERM_RED
# TERM_L_RED
# TERM_RED_LAVA
# TERM_RASPBERRY
# TERM_RED_RUST
# TERM_PINK
# TERM_ORANGE

#       
#       
#       

# Effects do not use the C ow W lines.  They use the one line below:
# X: depth : rarity : power : damage: gf type : timeout set : timeout rand
#
# depth and rarity are the same as the W line
# power is the effect or trap power/number
# damage is the damage per 100 levels for a smart trap - or damage for an effect cloud per 100 initial damage from the 
# gf type is the force type of the effect - numbers are hard coded in source 
# timeout set base time between effect instances 
# timeout rand is the random time between effects 

# All features have an implicit ' Default' action, which is actually a null
# transition (it doesn't do anything).
# This action can be changed using the DEFAULT action name.
# Several transitions to the same feature can be condensed with a single
# DEFAULT directive.

# Note: most passable features must have the BRIDGE flag ('F:' directive)
# to avoid abnormal tunnel termination. The most important exception is
# 'open floor'.
# It's not necessary to provide a BRIDGE action ('K:' directive) for most
# features, usually the implicit 'Default'action is enough, but it's
# strongly recommended that dangerous features should be bridged with harmless
# terrain. Sometimes we use explicit BRIDGE actions to add flavor.

# Note about the CAN_TUNNEL action: wall features shouldn't be turned into
# dangerous features after applying this action.

# Change chance is optional. Its values are specific to the transition.
